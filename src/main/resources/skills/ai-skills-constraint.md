# AI Skills Constraint - Swallowtail Butterfly Platform

## 概述

本约束文档定义了Swallowtail Butterfly AI平台的使用规范和约束条件。当子项目引入父POM时，将自动继承这些AI使用约束，确保AI功能的安全、可控和合规使用。

## 约束等级

### P0 - 强制约束 (必须遵守)
违反这些约束将导致系统拒绝服务或抛出异常。

### P1 - 推荐约束 (强烈建议遵守)
违反这些约束将记录警告日志，但不阻止服务执行。

### P2 - 可选约束 (建议遵守)
违反这些约束仅记录信息日志，用于监控和优化。

## P0 - 强制约束

### 1. 输入数据安全约束

#### 1.1 数据脱敏要求
```
约束ID: INPUT_DATA_SANITIZATION
级别: P0
描述: 所有输入数据必须进行脱敏处理
验证规则:
- 移除或掩码个人身份信息 (PII)
- 移除或掩码敏感商业信息
- 移除或掩码系统敏感信息
实施方式:
- 自动检测和警告未脱敏数据
- 强制要求数据预处理
```

#### 1.2 数据大小限制
```
约束ID: INPUT_DATA_SIZE_LIMIT
级别: P0
描述: 限制单次请求的数据大小
参数:
- 文本数据: 最大 10MB
- 图片数据: 最大 50MB
- 音频数据: 最大 100MB
- 视频数据: 最大 500MB
实施方式:
- 请求拦截器自动检查
- 超出限制直接拒绝
```

#### 1.3 数据格式验证
```
约束ID: INPUT_DATA_FORMAT_VALIDATION
级别: P0
描述: 验证输入数据的格式正确性
支持格式:
- 文本: UTF-8编码
- 图片: JPEG, PNG, WebP
- 音频: MP3, WAV, FLAC
- 视频: MP4, AVI, MOV
实施方式:
- 文件头检测
- MIME类型验证
```

### 2. 输出内容安全约束

#### 2.1 内容过滤
```
约束ID: OUTPUT_CONTENT_FILTERING
级别: P0
描述: 过滤AI生成的有害内容
过滤类型:
- 暴力内容
- 仇恨言论
- 色情内容
- 政治敏感内容
- 虚假信息
实施方式:
- 后处理内容过滤
- 关键词匹配
- 语义分析
```

#### 2.2 输出格式规范
```
约束ID: OUTPUT_FORMAT_STANDARDIZATION
级别: P0
描述: 标准化AI输出格式
要求:
- JSON格式输出
- 统一错误码
- 标准化响应结构
- 时间戳格式统一
```

### 3. 使用频率约束

#### 3.1 请求频率限制
```
约束ID: REQUEST_RATE_LIMITING
级别: P0
描述: 限制API调用频率
限制规则:
- 单个用户: 100次/分钟
- 单个IP: 500次/分钟
- 单个应用: 1000次/分钟
实施方式:
- 分布式限流
- 令牌桶算法
```

#### 3.2 资源使用限制
```
约束ID: RESOURCE_USAGE_LIMITING
级别: P0
描述: 限制计算资源使用
限制规则:
- CPU使用率: < 80%
- 内存使用率: < 85%
- GPU使用率: < 90%
- 磁盘I/O: < 100MB/s
```

### 4. 审计与日志约束

#### 4.1 操作审计
```
约束ID: OPERATION_AUDITING
级别: P0
描述: 记录所有AI操作
审计内容:
- 用户ID
- 操作时间
- 操作类型
- 输入参数摘要
- 输出结果摘要
- 处理耗时
- 错误信息
```

#### 4.2 安全日志
```
约束ID: SECURITY_LOGGING
级别: P0
描述: 记录安全相关事件
记录事件:
- 异常访问尝试
- 数据泄露风险
- 模型被篡改
- 配置变更
- 权限违规
```
